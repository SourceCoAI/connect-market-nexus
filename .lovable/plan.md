

# Plan: Auto-Ingest External Calculator Leads into Valuation Leads

## Problem
The `receive-valuation-lead` edge function currently only writes to `incoming_leads` (raw staging table). The Valuation Leads screen reads from `valuation_leads` which has structured columns. Leads from the external auto shop calculator never appear on screen.

## Solution
Update the edge function to **also insert into `valuation_leads`** by extracting structured fields from the JSONB `calculator_inputs` and `valuation_result` payloads. The `incoming_leads` row is kept as a raw audit trail.

## What Changes

### 1. Update `receive-valuation-lead` Edge Function
After upserting into `incoming_leads`, extract and map fields into a `valuation_leads` row:

**From `calculator_inputs` JSONB:**
- `revenue_ltm.value` → `revenue`
- `ebitda_ltm.value` → `ebitda`
- `service_type.value` → `industry` (e.g. "collision")
- `locations_count.value` → `locations_count`
- `owner_dependency.value` → `owner_dependency`
- `trend_24m.value` → `growth_trend`

**From `valuation_result` JSONB:**
- `businessValue.low/mid/high` → `valuation_low/mid/high`
- `qualityLabel.label` → `quality_label`
- `tier` → `quality_tier`
- `buyerLane.lane` → `buyer_lane`
- `narrative` → stored in `raw_valuation_results` for detail view
- `scoreBreakdown` → `raw_valuation_results`
- `propertyValue` → `calculator_specific_data`

**Direct fields from the POST body:**
- `full_name` → `full_name`
- `email` → `email`  
- `website` → `website` + `business_name` (extracted from domain)
- `region`, `city` → `region`, `location` (combined as "City, Region")
- `lead_source` → `lead_source`
- `external_lead_id` → `source_submission_id`

**Calculated:**
- `calculator_type` = `'collision'` (from `service_type` or hardcoded for this endpoint)
- `display_name` = auto-generated by DB trigger
- Store full `calculator_inputs` → `raw_calculator_inputs`
- Store full `valuation_result` → `raw_valuation_results`

Upsert on `email` in `valuation_leads` as well (same conflict strategy).

### 2. No UI Changes Required
The existing Valuation Leads table, filters, scoring, push-to-deals, and all actions already work with these structured columns. The lead will appear immediately after insertion with:
- Revenue, EBITDA, valuation range displayed in the table
- Quality badge, growth trend, owner dependency all populated
- Raw calculator inputs available via `raw_calculator_inputs` for detail views
- Scoring function (`calculate-valuation-lead-score`) will pick it up on next run

### 3. Deploy Updated Function
Redeploy `receive-valuation-lead` with the mapping logic.

## Technical Detail: Field Extraction

The extraction handles the nested structure where each calculator input is an object like:
```json
{ "value": 1400000, "label": "$1,400,000", "step_name": "Financials", ... }
```

We access `.value` for the actual data. For valuation results, the structure is flat objects like `businessValue.mid`.

The function will be resilient — any missing field simply maps to `null`, so partial payloads still create valid leads.
